# Docsify 側邊欄與 Markdown 教學

本教學將引導你了解 Markdown 的常用語法，並深入解析 Docsify 的側邊欄設定檔 `_sidebar.md` 的運作原理，最後提供各項語法的專屬練習題協助你鞏固所學。

---

## 1. Markdown 常用語法簡介

Markdown 是一種輕量級的標記語言，允許你使用易讀易寫的純文字格式編寫文件。以下是撰寫筆記與設定 Docsify 最常用到的語法指南：

### 📌 標題 (Headings)
在行首使用 `#` 加上一個空白來建立標題，`#` 的數量代表標題的層級（從 H1 到 H6）。
```markdown
# 一級標題（頁面大標題，建議一個檔案只用一個）
## 二級標題（主要段落標題）
### 三級標題（子段落標題）
```

### 📌 字體效果 (Text Styles)
* **粗體**：在文字前後加上兩個星號 `**`。
* *斜體*：在文字前後加上一個星號 `*`。
* ~~刪除線~~：在文字前後加上兩個波浪符號 `~~`。
```markdown
這是一段 **粗體字** 以進行強調。
這是一段 *斜體字*。
這是一段 ~~劃掉的文字~~。
```

### 📌 無序清單 (Unordered List)
使用星號 `*`、加號 `+` 或減號 `-` 開頭，後面加上一個空白。在側邊欄 `_sidebar.md` 中，清單是構建導覽選單的最重要語法。
```markdown
* 項目一
* 項目二
  * 子項目（縮排 2 個空白）
```

### 📌 有序清單 (Ordered List)
使用數字加上英文句點 `.`，再接一個空白即可建立有順序的清單。
```markdown
1. 第一步：下載 Git
2. 第二步：設定使用者名稱
3. 第三步：進行第一次 Commit
```

### 📌 超連結 (Hyperlink)
超連結的語法格式為 `[顯示文字](連結網址或相對檔案路徑)`。
* **中括號 `[]`**：填入畫面上要顯示的超連結文字。
* **小括號 `()`**：填入目標檔案的路徑或外部網址。
```markdown
[首頁](README.md)
[前往 Google](https://www.google.com)
```

### 📌 圖片 (Image)
圖片的語法和超連結非常像，只在最前面多了一個驚嘆號 `!`。格式為 `![替代文字](圖片路徑或網址)`。
```markdown
![我的專案 Logo](images/logo.png)
```

#### 💡 觀念釐清：相對路徑 vs 絕對路徑
在填寫超連結或圖片的 `()` 路徑時，選擇正確的路徑類型至關重要。這決定了你的網頁在本地瀏覽以及上傳到 GitHub 後是否能正常顯示。

| 類型 | 範例 | 說明 | 在 Docsify / 網頁中的表現 |
| :--- | :--- | :--- | :--- |
| **相對路徑** | `PolarPop.png`<br>`images/logo.png` | 以**當前檔案所在位置**為起點出發尋找目標檔案。 | **強烈推薦** ⭐<br>因為是相對位置，當你把整個專案資料夾上傳到 GitHub 或換到別台電腦時，連結與圖片都不會失效。 |
| **網路絕對路徑** | `https://img.icons8.com/.../cat.jpg` | 網路上資源的完整 URL 網址。 | **可以使用** 🌐<br>只要使用者有連接網路，就可以成功載入該圖片或開啟連結。 |
| **本機絕對路徑** | `D:\MyData\study\study-vault\docs\PolarPop.png` | 指向你個人電腦硬碟的完整檔案路徑（以 `C:\` 或 `D:\` 開頭）。 | **絕對不可使用** ❌<br>1. **安全限制**：瀏覽器出於安全機制，禁止網頁直接存取你本機硬碟的檔案。<br>2. **上傳失效**：一旦上傳到 GitHub，伺服器上根本沒有你的 `D槽`，所有人（包括你自己）都會看到破圖。 |

---

### 📌 引用區塊 (Blockquotes)
在行首使用大於符號 `>`，可用於補充說明、提示、警告或名言引用。
```markdown
> 💡 提示：在 Docsify 中，側邊欄的檔案路徑是以 `docs` 資料夾作為根目錄的相對路徑。
```

### 📌 程式碼 (Code)
* **行內程式碼 (Inline Code)**：使用一組反單引號 `` ` ``（鍵盤上 Tab 鍵上方那個按鍵）包圍，用於標示指令、變數或檔名。
* **程式碼區塊 (Code Block)**：使用兩組三個反單引號 ` ``` ` 包圍，並可在開頭指定程式語言名稱（如 `html`, `css`, `javascript`, `bash`）以啟用語法色彩。

````markdown
請在終端機輸入 `git status` 查看狀態。

```javascript
const name = "Antigravity";
console.log(`Hello, ${name}!`);
```
````

## 2. 解析 `_sidebar.md`

在 Docsify 中，側邊欄是用來導覽整個網站的目錄。我們來看看你目前的 [_sidebar.md](file:///d:/MyData/study/study-vault/docs/_sidebar.md) 內容：

```markdown
* [首頁](README.md)

* [關於我](about.md)

* 🛠️常用工具
  * [Git基礎指令](git基礎指令.md)
  * [GitHub Pages美化](GitHub_Pages_美化指南.md)

* 📚 學習筆記
  * [Docsify側邊欄與Markdown教學](Docsify側邊欄與Markdown教學.md)
  * [0710御三家](0710御三家.md)
  * [0710手機晶片大廠](0710手機晶片大廠.md)
  * [0710Git 基礎知識](0710GIT的基礎知識.md)
  * [0711公司](0711公司.md)
```

### 結構詳細解析：

1. **獨立連結**：`* [關於我](about.md)`
   * 點擊後直接切換到該檔案。它屬於第一層目錄。

2. **分類組標題**：`* 🛠️常用工具` 與 `* 📚 學習筆記`
   * 注意這兩行沒有 `[]()` 超連結。這在 Docsify 中代表**靜態分組標題**，滑鼠點擊不會切換頁面，僅用來歸納底下的子項目。

3. **縮排的子項目**：
   ```markdown
     * [Git基礎指令](git基礎指令.md)
     * [GitHub Pages美化](GitHub_Pages_美化指南.md)
   ```
   * 在項目開頭**多縮排兩個空白**，Docsify 就會知道這兩個檔案屬於「🛠️常用工具」底下。這會在網頁側邊欄形成漂亮的展開/收合樹狀結構。

## 3. 側邊欄運作的重要觀念

* **路徑問題**：側邊欄中的路徑都是相對於 `docs` 目錄的相對路徑。
* **index.html 設定**：必須在 `index.html` 的設定中開啟 `loadSidebar: true`，側邊欄才會生效。
* **清單符號的一致性**：在 `_sidebar.md` 中，清單可以使用 `*`、`+`、`-`，但在同一個檔案中**統一使用一種符號（如 `*`）**可以避免排版混亂，程式碼也更整潔。
